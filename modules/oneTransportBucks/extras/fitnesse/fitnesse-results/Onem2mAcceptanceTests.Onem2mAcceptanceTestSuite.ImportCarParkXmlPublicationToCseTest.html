A car park situation record imported into one2m creates an appropriate structure in the CSE


A Buckinghamshire County Council car park situation record imported by a data feed creates the following structure in the CSE


Given a car park situation record imported from BCC:

Car Park Situation Record
car park identity fillRate exitRate totalCapacity latitude longitude imported?
BUCK-Walton Street MSCP 0 0 525 51.8122864 -0.809660852 true

And an Application Entity in the CSE:

Ae Configuration
name configured?
C-BCCCarParkFeedImport true


Then the following containers and values should be created in the CSE:

Query:Check Cse Container Json Content
Container Path Content Value Json Matches
BCCCarParkFeedImport/BUCK-Walton_Street_MSCP { "carParkIdentity": "BUCK-Walton_Street_MSCP", "fillRate": "0", "exitRate": "0", "totalCapacity": "525", "latitude": "51.8122864", "longitude": "-0.809660852" } true


And for data discovery, the XML tag names should be added as label to the containers in the CSE:

Query:Label Query Handler
Container Path Labels Contain Result
BCCCarParkFeedImport/BUCK-Walton_Street_MSCP datex2:carPark true
